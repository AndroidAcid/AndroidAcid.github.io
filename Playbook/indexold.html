02 // First 30 Minutes

EVERY NEW APK

Run this before anything else. Two Criticals before touching the device is not unusual.

bash — androidacid initial triage
~/target $ adb shell pm path com.target.app
package:/data/app/com.target.app-1/base.apk
~/target $ adb pull /data/app/com.target.app-1/base.apk ./target.apk
[ ✓ ] 18.4 MB pulled
~/target $ apktool d target.apk -o src/ && jadx -d jadx/ target.apk
[ ✓ ] 847 Java classes extracted
 
# Phase 1: Manifest — highest ROI file, read it first
~/target $ grep -n 'exported="true"\|debuggable\|allowBackup\|cleartext' src/AndroidManifest.xml
L12: android:debuggable="true" ← Critical (RCE via jdwp)
L31: android:exported="true" (.ui.AdminActivity) ← High (auth bypass)
L44: android:allowBackup="true" ← Medium (data extraction)
 
# Phase 2: Secrets scan

03 // Contribute

ADD A NEW PAGE

Every guide is a Markdown file in /Android/ on the repo plus one new card on this page. Here is the exact workflow — takes about 10 minutes to wire up a new guide.

01

Create the .md file

Add your file inside /Android/ in the repo.

Naming convention:
android-[topic]-guide.md

Examples:
android-frida-guide.md
android-deeplink-guide.md
android-network-guide.md
android-apk-patching-guide.md

02

Follow the content structure

Mirror the existing guide format (see template below):

## Overview — one paragraph
## Phase 0 — Setup — tools + install
## Phase N — [Topic] — repeat per attack surface. Each phase needs: grep command, vulnerable code snippet, PoC, impact statement.
## Severity Reference — table
## Validation Gate — checklist
## Tool Quick Reference — table

03

Add a card on this page

Open index.html, find <!-- GUIDE 0N --> and copy the card block above it. Update:

href → your new filename
.card-num → next number
.card-sev → sev-c / sev-h / sev-m
h3 → title in caps
p → 2–3 sentence description
.card-preview → 4–5 representative commands
.card-tags → tool names

04

Update the stats counter

In index.html, find the stats strip near the top. The Guides counter uses data-target="4" — increment by 1.

Also update the Phases and PoC counts if your guide adds more.

Commit and push to main → GitHub Pages deploys automatically within ~60 seconds.

TOOL QUICK REF

⬡ APK Prep

grep -n 'exported="true"' AndroidManifest.xml

grep -n 'uses-permission' AndroidManifest.xml
grep -n 'cleartext\|networkSecurityConfig' *.xml

⬡ Secrets

grep -rn 'AKIA[0-9A-Z]{16}' jadx/
grep -rn 'AIza[0-9A-Za-z_-]{35}' jadx/
